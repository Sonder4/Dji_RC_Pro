# 头脑风暴：RustDesk 融合方案 (已定稿)

## 1. 核心目标
在现有 DJI RC Pro App 中增加“融合”功能，使其能够通过局域网直连观看远程电脑画面，同时保持 App 原有的后台控制逻辑（UDP 摇杆数据传输）。

## 2. 方案总结
采用 **Flutter Embedding (嵌入式)** 方案，将 RustDesk 编译为 AAR 库集成到 Android 原生项目中。

### 2.1 界面流
*   **第一界面 (MainScreen)**:
    *   保持现有功能。
    *   新增“投屏配置”：输入目标 IP 和访问密码。
    *   新增“进入投屏”按钮：点击跳转第二界面。
*   **第二界面 (RustDeskScreen)**:
    *   **全屏横屏**显示远程画面。
    *   **交互**: 默认**无**键鼠控制（仅观看），保留手势缩放 (Pinch-to-zoom)。
    *   **工作台 (Toolbar)**: 保留，用于打开终端、摄像头、设置。
*   **第三界面 (Terminal/Camera)**:
    *   **终端**: 通过 Toolbar 打开，支持软键盘和快捷键 (Esc, Ctrl+C)。
    *   **摄像头**: 通过 Toolbar 打开，查看电脑端 USB 摄像头，支持切换。

### 2.2 功能裁剪与保留
| 功能模块 | 状态 | 说明 |
| :--- | :--- | :--- |
| **账户系统** | ❌ 移除 | 不显示登录/注销入口。 |
| **ID/中继服务器** | ✅ 保留 ||
| **Socks5/Http代理** | ❌ 移除 | 简化网络配置。 |
| **自动更新** | ❌ 移除 | 避免 App 内弹窗。 |
| **WebSocket** | ✅ 保留 | 核心协议组件，必须保留。 |
| **终端 (Terminal)** | ✅ 保留 | 用于远程命令行操作。 |
| **远程摄像头** | ✅ 保留 | 查看 PC 端 USB 摄像头。 |
| **文件传输** | ❌ 隐藏 | 本次需求暂不需要。 |

### 2.3 技术策略
*   **连接方式**: 局域网 IP 直连 + 固定密码。
*   **重连机制**: 断开后自动重试 15 次，失败后退回第一界面。
*   **性能**: 启用 MediaCodec 硬解码，禁用音频传输。
*   **后台**: 确保 `UdpService` 在投屏期间持续运行。

## 3. 下一步计划
1.  修改 RustDesk 源码 (`rustdesk/flutter`) 实现上述裁剪。
2.  编译 RustDesk AAR。
3.  在 Android 项目中集成 AAR 并实现 Navigation 跳转。
