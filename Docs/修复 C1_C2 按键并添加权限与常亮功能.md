# 任务计划：修复 C1/C2 按键并添加系统功能

## 1. 识别并修复 C1/C2 按键
- **增加 HID 调试日志**：修改 `UsbHidProtocolParser.kt`，在 `parse` 函数中增加对原始数据的 HEX 打印，以便用户通过 Logcat 观察按下 C1/C2 时的变化。
- **监控系统键值**：在 `MainActivity.kt` 的 `onKeyDown` 中增加日志打印，捕获所有未识别的按键 `keyCode`，确认 C1/C2 是否走的是系统标准输入。
- **更新协议解析**：一旦通过日志确认了按键所在的字节和位，立即更新 `UsbHidProtocolParser` 中的 `buttonMask` 映射。

## 2. 添加权限申请
- **修改 AndroidManifest.xml**：
    - 添加蓝牙权限：`BLUETOOTH_SCAN`, `BLUETOOTH_CONNECT`, `BLUETOOTH_ADMIN`。
    - 添加 WiFi 权限：`ACCESS_WIFI_STATE`, `CHANGE_WIFI_STATE`, `ACCESS_FINE_LOCATION`（WiFi 扫描必需）。
- **优化权限请求逻辑**：更新 `MainActivity.kt` 中的 `requestPermissions` 函数，将新增的权限加入动态申请列表。

## 3. 实现屏幕常亮
- **修改 MainActivity.kt**：在 `onCreate` 方法中添加 `FLAG_KEEP_SCREEN_ON` 标志位，确保 App 在前台时屏幕保持开启。

## 4. 验证与编译
- 重新执行 `assembleDebug` 并安装，提示用户查看日志并进行按键测试。
